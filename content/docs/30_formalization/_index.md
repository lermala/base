## Что такое моделирование?

**Модель** — это структурированное представление информации о системе, созданное с целью анализа, проектирования или коммуникации, и обладающее определённым формальным или полуструктурным языком (графическим, табличным, текстовым).

Иными словами:
- у модели есть **структура** (формат, который можно воспроизводить и интерпретировать),
- она **отображает часть реальности**,
- и **служит инструментом мышления**, а не просто хранением фактов.

А **моделирование** — это процесс создания таких моделей.

{{< callout >}}
  Модель ≠ реальность.
  Она лишь упрощает реальность, **выделяя существенные свойства и игнорируя несущественные**.
{{< /callout >}}

### Виды моделирования

Есть разные виды моделей и используют их в самых разных отраслях:

| Вид моделирования                   | Что описывает                             | Примеры                                                      |
| ----------------------------------- | ----------------------------------------- | ------------------------------------------------------------ |
| **Физическое**                      | Реальные объекты в материальном виде      | макет здания, прототип устройства                            |
| **Математическое**                  | Описывает количественные зависимости | формулы, модели прогнозирования |
| **Логическое**                      | Отображает порядок действий и условия           | блок-схемы, псевдокод                                  |
| **Вербальное (словесное)**          | Описывает объект словами             | технические требования, сценарии use case                    |
| **Графическое (визуальное)**        | Представляет информацию схемами и диаграммами     | ментальные карты, архитектурные схемы                        |
| **Табличное**        | Показывает связи и зависимости в виде таблиц     | матрицы (ролей, решений, трассируемости)                        |
| **Имитационное**                    | Воспроизводит поведение системы во времени | моделирование движения транспорта, нагрузки серверов         |

## Моделирование в IT

В контексте IT чаще всего говорят о:
- **Бизнес-моделировании** (BPMN, IDEF0, eEPC)
- **Моделировании требований** (RML, UML Use Case, SysML)
- **Моделировании архитектуры** (C4, UML Component/Deployment)
- **Моделировании данных** (ER, Class, State)
- **Моделировании взаимодействий** (Sequence, Communication)

Есть много разных нотаций, тут подробно о них писать не буду. О некоторых написала в отдельных страничках:
- [BPMN](/docs/30_formalization/bpmn/)

- [ER](/docs/40_db/erd/)
